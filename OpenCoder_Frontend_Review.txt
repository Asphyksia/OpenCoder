================================================================================
PROBLEMAS CRÍTICOS ❌
================================================================================

1. VULNERABILIDADES DE SEGURIDAD EN DEPENDENCIAS
   ----------------------------------------------
   Severidad: CRÍTICA
   
   Al ejecutar npm install, se detectaron:
   - 7 vulnerabilidades (3 moderate, 3 high, 1 critical)
   - Next.js 14.2.21 tiene una vulnerabilidad de seguridad documentada
     Ver: https://nextjs.org/blog/security-update-2025-12-11
   
   ACCIÓN REQUERIDA:
   - Actualizar Next.js a la versión más reciente: npm install next@latest
   - Ejecutar: npm audit fix --force

2. TSCONFIG.JSON FALTA BASEURL
   ----------------------------
   Severidad: ALTA
   
   El archivo tsconfig.json tiene paths configurados pero falta baseUrl:
   
   ACTUAL:
   {
     "compilerOptions": {
       "paths": {
         "@/*": ["./src/*"]
       }
     }
   }
   
   CORRECTO:
   {
     "compilerOptions": {
       "baseUrl": ".",
       "paths": {
         "@/*": ["./src/*"]
       }
     }
   }
   
   IMPACTO: TypeScript no puede resolver los imports con alias @/
   Esto causa 40+ errores de compilación.

3. TIPO INCORRECTO EN USEHEALTHCHECK
   ----------------------------------
   Severidad: MEDIA
   
   Archivo: src/hooks/useStatus.ts:61
   
   PROBLEMA:
   return {
     isHealthy: healthQuery.data?.status === "ok",
   };
   
   HealthResponse define status como string, no como literal "ok":
   
   SOLUCIÓN:
   Opción A: Cambiar HealthResponse para usar literal:
     export interface HealthResponse {
       status: "ok" | "healthy";
       version: string;
     }
   
   Opción B: Cambiar la comparación:
     isHealthy: healthQuery.data?.status === "healthy"

dependencias vulnerables, el proyecto debería compilar sin errores y estar
listo para desarrollo activo.
