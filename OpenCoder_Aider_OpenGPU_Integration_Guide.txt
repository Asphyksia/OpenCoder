                           1. RESUMEN EJECUTIVO
================================================================================

PROBLEMA ACTUAL:
- SimpleAgentEngine solo hace chat básico, NO usa las capacidades de Aider
- AgentEngine está diseñada para Aider pero falla por configuración incorrecta
- La URL de OpenGPU usada anteriormente causaba redirecciones que litellm no
  manejaba correctamente

CAUSA RAÍZ:
- URL incorrecta: https://relay.opengpu.network/v1 (redirige)
- URL correcta: https://relaygpu.com/backend/openai/v1 (directa)

SOLUCIÓN:
- Usar la URL directa del backend de OpenGPU
- Configurar variables de entorno ANTES de importar Aider
- Usar el prefix "openai/" en el nombre del modelo

================================================================================
                        2. ARQUITECTURA DE OPENGPU
================================================================================

OpenGPU Relay soporta DOS métodos de autenticación:

MÉTODO 1: X-API-Key (nativo)
-----------------------------
curl -X POST https://relay.opengpu.network/v2/ollama/api/chat \
  -H "X-API-Key: relay_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-oss:20b", "messages": [...]}'

MÉTODO 2: Bearer Token (compatible con OpenAI/Anthropic SDK)
-------------------------------------------------------------
curl -X POST https://relaygpu.com/backend/openai/v1/chat/completions \
  -H "Authorization: Bearer relay_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"model": "openai/gpt-5.2", "messages": [...]}'

IMPORTANTE: Si ambos headers están presentes, X-API-Key tiene prioridad.
           Aider/litellm SOLO envía "Authorization: Bearer", por lo que no
           hay conflicto.

"""
Agent Engine - Aider Wrapper for OpenCoder (Versión Corregida)

Este módulo proporciona un wrapper alrededor de Aider configurado
para trabajar con OpenGPU Relay.

IMPORTANTE: La configuración de variables de entorno DEBE hacerse antes
de cualquier import de Aider.
"""

import os
import io
import asyncio
from pathlib import Path
from typing import Any, Optional
from dataclasses import dataclass, field
from loguru import logger

# ============================================================================
# CONFIGURACIÓN CRÍTICA: Debe ejecutarse ANTES de importar Aider
# ============================================================================

def configure_opengpu_for_aider():
    """
    Configura las variables de entorno necesarias para que Aider/litellm
    funcione con OpenGPU Relay.
    
    Debe llamarse ANTES de cualquier import de Aider.
    """
    api_key = os.getenv("OPENGPU_API_KEY", "")
    base_url = os.getenv("OPENGPU_BASE_URL", "https://relaygpu.com/backend/openai/v1")
    
    if api_key:
        # Variables para OpenAI SDK (usado por Aider internamente)
        os.environ["OPENAI_API_KEY"] = api_key
        os.environ["OPENAI_API_BASE"] = base_url
        
        # Variables para litellm (usado por Aider para modelos no-OpenAI)
        os.environ["LITELLM_API_KEY"] = api_key
        os.environ["LITELLM_BASE_URL"] = base_url
        
        logger.info(f"[OpenCoder] Configurado para OpenGPU: {base_url}")
    else:
        logger.warning("[OpenCoder] No se encontró OPENGPU_API_KEY")

# Ejecutar configuración ANTES de importar Aider
configure_opengpu_for_aider()

# ============================================================================
# IMPORTS DE AIDER (después de configuración)
# ============================================================================

AIDER_AVAILABLE = False
try:
    from aider.coders import Coder
    from aider.io import InputOutput
    from aider.models import Model
    AIDER_AVAILABLE = True
    logger.info("[OpenCoder] Aider cargado correctamente")
except ImportError as e:
    Coder = None
    InputOutput = None
    Model = None
    logger.warning(f"[OpenCoder] Aider no disponible: {e}")


# ============================================================================
# DATA CLASSES
# ============================================================================

@dataclass
class FileChange:
    """Representa un cambio en un archivo."""
    filename: str
    diff: str
    operation: str = "modified"  # created, modified, deleted


@dataclass
class AgentEvent:
    """Representa un evento durante la ejecución del agente."""
    event_type: str
    content: str
    timestamp: str = field(default_factory=lambda: "")


@dataclass
class AgentResponse:
    """Respuesta estructurada del agente."""
    success: bool
    message: str
    events: list[AgentEvent] = field(default_factory=list)
    file_changes: list[FileChange] = field(default_factory=list)
    diffs: str = ""
    error: Optional[str] = None


# ============================================================================
# AGENT ENGINE (VERSIÓN CORREGIDA)
# ============================================================================

class AgentEngine:
    """
    Wrapper alrededor de Aider Coder para uso programático.
    
    Esta clase usa Aider con OpenGPU Relay para proporcionar capacidades
    de edición de código asistida por IA.
    
    Ejemplo de uso:
        >>> engine = AgentEngine("./mi-proyecto", "openai/gpt-4o")
        >>> result = await engine.execute("Añade función hello_world")
        >>> print(result.message)
    """
    
    def __init__(
        self,
        repo_path: str,
        model_name: str = "openai/gpt-4o",
        read_only: bool = False,
        auto_commits: bool = True,
    ) -> None:
        """
        Inicializa el Agent Engine.
        
        Args:
            repo_path: Ruta al repositorio de código.
            model_name: Nombre del modelo con prefix (ej: "openai/gpt-4o").
            read_only: Si True, no hace cambios (dry-run).
            auto_commits: Si True, hace commits automáticos.
        """
        self.repo_path = Path(repo_path).resolve()
        self.model_name = model_name
        self.read_only = read_only
        self.auto_commits = auto_commits
        
        self._coder: Optional[Coder] = None
        self._events: list[AgentEvent] = []
        self._file_changes: list[FileChange] = []
        self._output_capture: io.StringIO = None
        
        # Asegurar que el directorio existe
        if not self.repo_path.exists():
            self.repo_path.mkdir(parents=True, exist_ok=True)
    
    def _setup_coder(self) -> Coder:
        """Configura y retorna la instancia de Aider Coder."""
        if self._coder is not None:
            return self._coder
        
        if not AIDER_AVAILABLE:
            raise RuntimeError("Aider no está disponible. Instala con: pip install aider-chat")
        
        # Crear el modelo - el prefix "openai/" es OBLIGATORIO
        # Esto le dice a Aider que use OPENAI_API_BASE para las llamadas
        model = Model(self.model_name)
        
        # Configurar captura de output
        self._output_capture = io.StringIO()
        aider_io = InputOutput(
            pretty=False,
            fancy_input=False,
            input=io.StringIO(),
            output=self._output_capture,
        )
        
        # Buscar archivos Python en el repo
        fnames = list(self.repo_path.glob("*.py")) if self.repo_path.exists() else []
        
        # Crear el Coder
        self._coder = Coder.create(
            main_model=model,
            io=aider_io,
            fnames=[str(f) for f in fnames[:10]],  # Limitar a 10 archivos
            auto_commits=self.auto_commits,
            verbose=False,
        )
        
        return self._coder
    
    async def execute(self, user_message: str) -> AgentResponse:
        """
        Ejecuta un mensaje del usuario a través del agente.
        
        Args:
            user_message: El mensaje/comando del usuario.
        
        Returns:
            AgentResponse con los resultados estructurados.
        """
        self._events = []
        self._file_changes = []
        
        try:
            self._add_event("system", "Inicializando agente...")
            
            coder = self._setup_coder()
            self._add_event("system", f"Aider Coder inicializado con modelo: {self.model_name}")
            
            self._add_event("user", user_message)
            self._add_event("thinking", "Procesando solicitud...")
            
            # Ejecutar el mensaje a través de Aider
            coder.run(user_message)
            
            # Capturar output
            output = self._output_capture.getvalue() if self._output_capture else ""
            
            # Parsear eventos del output
            self._parse_output(output)
            
            # Capturar cambios de archivos
            self._capture_file_changes(coder)
            
            # Construir diffs
            diffs_text = "\n".join([fc.diff for fc in self._file_changes])
            
            return AgentResponse(
                success=True,
                message=f"Procesado: {user_message[:50]}...",
                events=self._events,
                file_changes=self._file_changes,
                diffs=diffs_text,
                error=None
            )
            
        except Exception as e:
            error_msg = str(e)
            self._add_event("error", error_msg)
            return AgentResponse(
                success=False,
                message="Error durante la ejecución",
                events=self._events,
                file_changes=self._file_changes,
                diffs="",
                error=error_msg
            )
    
    def _add_event(self, event_type: str, content: str) -> None:
        """Añade un evento a la lista."""
        import time
        self._events.append(AgentEvent(
            event_type=event_type,
            content=content,
            timestamp=str(time.time())
        ))
    
    def _parse_output(self, output: str) -> None:
        """Parsea el output de Aider y categoriza eventos."""
        if not output:
            return
        
        for line in output.split("\n"):
            line = line.strip()
            if not line:
                continue
            
            if "git" in line.lower() or "commit" in line.lower():
                self._add_event("git", line)
            elif "file" in line.lower() and any(w in line.lower() for w in ["created", "modified", "deleted"]):
                self._add_event("file", line)
            elif "error" in line.lower():
                self._add_event("error", line)
            else:
                self._add_event("output", line)
    
    def _capture_file_changes(self, coder: Coder) -> None:
        """Captura los cambios de archivos del Coder."""
        try:
            if hasattr(coder, 'repo') and coder.repo:
                repo = coder.repo
                if hasattr(repo, 'get_tracked_files'):
                    for filepath in repo.get_tracked_files():
                        self._file_changes.append(FileChange(
                            filename=str(filepath),
                            diff=f"[Archivo modificado: {filepath}]",
                            operation="modified"
                        ))
        except Exception:
            pass
    
    def get_status(self) -> dict:
        """Retorna el estado actual del repositorio."""
        return {
            "repo_path": str(self.repo_path),
            "model": self.model_name,
            "read_only": self.read_only,
            "events_count": len(self._events),
            "file_changes_count": len(self._file_changes),
            "aider_available": AIDER_AVAILABLE,
        }
    
    def reset(self) -> None:
        """Resetea el estado del engine."""
        self._events = []
        self._file_changes = []
        self._output_capture = None


# ============================================================================
# SIMPLE AGENT ENGINE (para chat básico sin edición)
# ============================================================================

class SimpleAgentEngine:
    """
    Motor simple para chat básico sin capacidades de edición de Aider.
    
    Usar solo cuando no se necesite edición de código.
    """
    
    def __init__(self, repo_path: str, model_name: str = "openai/gpt-4o") -> None:
        self.repo_path = Path(repo_path)
        self.model_name = model_name
        self._history: list[dict] = []
    
    async def chat(self, message: str) -> dict:
        """Envía un mensaje y recibe respuesta (sin edición de código)."""
        from openai import AsyncOpenAI
        
        self._history.append({"role": "user", "content": message})
        
        client = AsyncOpenAI(
            base_url=os.environ.get("OPENAI_API_BASE", "https://relaygpu.com/backend/openai/v1"),
            api_key=os.environ.get("OPENAI_API_KEY", ""),
        )
        
        # Extraer nombre del modelo sin el prefix "openai/"
        model_name = self.model_name.replace("openai/", "")
        
        response = await client.chat.completions.create(
            model=model_name,
            messages=self._history,
        )
        
        assistant_message = response.choices[0].message.content
        self._history.append({"role": "assistant", "content": assistant_message})
        
        return {
            "success": True,
            "message": assistant_message,
            "events": [{"event_type": "chat", "content": "Respuesta generada"}],
            "error": None
        }

CAMBIOS NECESARIOS:

1. En get_or_create_engine(), cambiar:

   ANTES:
   --------
   self.engines[session_id] = SimpleAgentEngine(
       repo_path=repo_path,
       model=model
   )

   DESPUÉS:
   ---------
   self.engines[session_id] = AgentEngine(
       repo_path=repo_path,
       model_name=model_name,  # Asegurar que incluya prefix "openai/"
       read_only=read_only
   )

2. En el endpoint /chat, usar execute() en lugar de chat():

   ANTES:
   --------
   result = await engine.chat(request.message)

   DESPUÉS:
   ---------
   result = await engine.execute(request.message)

================================================================================
                     10. NOTAS IMPORTANTES
================================================================================

1. ORDEN DE IMPORTS: La configuración de variables de entorno DEBE ejecutarse
   antes de cualquier import de Aider. Esto es crítico porque Aider/litellm
   leen las variables al importarse.

2. PREFIX DEL MODELO: El prefix "openai/" es OBLIGATORIO. Sin él, Aider
   intentará conectar a api.openai.com en lugar del endpoint configurado.

3. URL CORRECTA: Usar https://relaygpu.com/backend/openai/v1 NO usar
   https://relay.opengpu.network/v1 (esta última redirige y causa problemas).

4. HEADERS: OpenGPU acepta tanto "Authorization: Bearer" como "X-API-Key".
   Aider/litellm usa "Authorization: Bearer" que funciona correctamente.

5. MODELOS: Para ver modelos disponibles:
   curl https://relay.opengpu.network/v2/models
