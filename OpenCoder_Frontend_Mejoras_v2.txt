2.1 CÓDIGO Y SYNTAX HIGHLIGHTING
--------------------------------
PRIORIDAD: Alta

Problema: El código se muestra con syntax highlighting básico, pero los bloques
dentro del chat no tienen el formato rico que debería.

Mejoras:
- Añadir detección automática del lenguaje por extensión de archivo
- Colorear el código en los diffs
- Añadir tema personalizado que coincida con el tema de la app

Ejemplo de implementación:
```tsx
// Detectar lenguaje por nombre de archivo
function detectLanguage(filename: string): string {
  const ext = filename.split('.').pop()?.toLowerCase()
  const langMap: Record<string, string> = {
    'py': 'python',
    'js': 'javascript',
    'ts': 'typescript',
    'tsx': 'tsx',
    'jsx': 'jsx',
    'json': 'json',
    'md': 'markdown',
    'css': 'css',
    'html': 'html',
    'sh': 'bash',
    'yaml': 'yaml',
    'yml': 'yaml',
  }
  return langMap[ext || ''] || 'text'
}
```

---------------------------

Problema: Los timestamps (22:11:05, 17:17:45) son difíciles de leer y
se ven pequeños.

import { formatDistanceToNow } from 'date-fns'
import { es } from 'date-fns/locale'

function formatRelativeTime(timestamp: string): string {
  const date = new Date(timestamp)
  return formatDistanceToNow(date, { addSuffix: true, locale: es })
}

```tsx
// Error boundary para el chat
class ChatErrorBoundary extends React.Component {
  state = { hasError: false }
  
  static getDerivedStateFromError(error) {
    return { hasError: true }
  }
  
  componentDidCatch(error, info) {
    console.error('Chat error:', error, info)
    // Enviar a servicio de monitoreo
  }
  
  render() {
    if (this.state.hasError) {
      return (
        <div className="p-4 text-center">
          <AlertCircle className="w-8 h-8 mx-auto text-red-500 mb-2" />
          <p>Something went wrong</p>
          <Button onClick={() => this.setState({ hasError: false })}>
            Retry
          </Button>
        </div>
      )
    }
    return this.props.children
  }
}
```
